# DSA Patterns

Keeping track of DSA problems organized by pattern instead of dumping them by random difficulty or platform. The whole point is that most interview questions aren't actually unique - they're a variation of one of a handful of patterns, and recognizing which pattern a problem belongs to is most of the actual skill being tested.

Started this because I'd solve something on LeetCode, feel good about it, then completely blank on a similar problem two weeks later because I never connected the two as being the same idea wearing a different costume. This repo is me trying to fix that.

## How this is organized

Each pattern gets its own folder with:
- A README explaining when to spot that pattern, plus a checklist of problems in it
- The actual solutions, named like `01-problem-name.java`, with a comment block on top (problem, pattern, my approach, time/space complexity)

Folders are currently just stubs with the problem list pre-filled in, since I'm structuring this out before actually solving everything. I'll fill in real code and check things off the list as I go, not all at once.
