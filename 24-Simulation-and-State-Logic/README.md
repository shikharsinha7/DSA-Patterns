# Simulation / Step-by-Step State Logic

**When to use:** Problems that describe a process (a game, a robot, a sequence of digit operations) and test whether you can translate that process into clean code exactly - without over-engineering it into a fancier algorithm, and without missing edge cases like overflow or cycles.

Smaller folder than the other two - these show up less often, but they're a distinct enough flavor (state tracking over time/steps, rather than pure array/string index logic) to keep separate.
