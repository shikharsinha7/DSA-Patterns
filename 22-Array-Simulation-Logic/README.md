# Array Simulation / In-place Logic

**When to use:** No named data structure (hashmap, stack, heap) actually helps here. These are pure index-math, swap, and loop-invariant problems - the "aha" isn't pattern recognition, it's careful, disciplined implementation under space/order constraints.

Difference from the other patterns I keep forgetting: Two Pointers and Sliding Window still have a clear "shape" you recognize by name. These don't - the interviewer picks them specifically to see if you can reason cleanly without falling back on a memorized trick.
