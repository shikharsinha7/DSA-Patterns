# String Simulation / In-place Logic

**When to use:** Same spirit as Array Simulation, applied to strings. No regex shortcuts, no library parsing tricks - just careful index math and in-place transformation. Different from `19-String-Matching`, which is about *searching* for a pattern (KMP, Rabin-Karp); this folder is about *constructing or transforming* a string correctly under tight space constraints.
